Welcome to the AgroFlow documentaion

AgroFlow is an internal application developed for the Agro Mousquetaires group. Its goal is to optimize project manager and automation engineer workflows by automating and centralizing the management of TIA Portal projects using the Openness API.

This documentation presents the architecture, objectives, main modules, and overall usage of AgroFlow. It serves as an entry point for all users and maintainers of the application.

Context and Objectives

The AgroFlow project addresses the growing need for integration between IT (Information Technology) and OT (Operational Technology) systems in the food industry. It enables:

  • Easy interconnection between IT/OT teams and streamlined data exchanges.

  • Centralized management of Agro Mousquetaires TIA Portal projects and simplified configuration.

  • Automation of recurring tasks (import/export, code generation, diagnostics) using robust scripts.

  • Reduced implementation time, reliable deployments, and guaranteed compliance with internal standards.

Architecture and Main Modules

The application consists of several major modules:

  • Main Interface: Centralized access management, function navigation, user/admin session management, dynamic TIA Portal version selection.

  • Diagnostic Assistance: Automated generation of BLKJump variables in the diagnostic FC for quick and reliable diagnostics.

  • Project Reception: Seamless integration of new projects, compliance checks (LTU library), and automated initial configuration.

  • Dynamic TIA Portal Version Management: Enables usage of multiple TIA Portal versions without changing applications, through smart Openness DLL loading.

Key classes include: ExploreTIA, PlcGenerateTiaCode, HMATIAOpenness, TIAVersionManager, and TIAAssemblyLoader.

Technical Prerequisites

  • TIA Portal version 17 or higher, with Openness license enabled.

  • AgroFlow application installed on the workstation.

  • TIA project compliant with Agro Mousquetaires standards, including the LTU library.

  • Windows 10/11, .NET 7.0, and Windows Forms (.NET 4.8) for the user interface.

Key Features

  • Time savings and simplicity: A single tool for all TIA Portal versions, automation of repetitive tasks, centralized management.

  • Intuitive interface: Fast navigation, user profile management, direct access to main functions (diagnostics, import/export, logs, etc.).

  • Robustness and scalability: Modular architecture, dynamic TIA version management, easy addition of new modules.
