      InterfaceMain

      Le module Interface Principale d’AgroFlow constitue le point d’entrée graphique de l’application. Il centralise l’accès à toutes les fonctionnalités clés, assure la gestion multi-utilisateur (admin/utilisateur standard) et permet une navigation rapide et intuitive entre les modules métier de la solution. Cette documentation présente la structure, le fonctionnement et l’architecture technique de l’interface, conçue pour répondre aux besoins des automaticiens et chefs de projet du groupe Agro Mousquetaires.

      Présentation et Objectifs

      L’interface principale vise à :

      • Offrir une expérience utilisateur moderne et adaptée aux standards Agro Mousquetaires

      • Centraliser l’accès aux fonctions critiques (diagnostic, réception de projet, etc.)

      • Gérer efficacement les droits d’accès (distinction admin/utilisateur standard)

      • Garantir une navigation fluide entre les modules métiers

      • Historiser toutes les actions importantes via un module de log accessible

      Fonctionnement Général

      L’interface principale AgroFlow repose sur plusieurs principes de fonctionnement :

      • Interface graphique Windows Forms personnalisée : Respecte la charte graphique Agro Mousquetaires, avec menu principal rétractable/étirable et gestion avancée de la fenêtre (redimensionnement, déplacement, maximisation).

      • Gestion des sessions : Distinction claire entre utilisateur standard et administrateur, adaptation dynamique des fonctionnalités disponibles selon le statut.

      • Sélection dynamique de la version TIA Portal : L’utilisateur choisit la version installée, mémorisée entre les sessions.

      • Navigation modulaire : Chaque fonction métier s’ouvre sous forme de UserControl pour garantir clarté et rapidité d’accès.

      • Gestion centralisée des logs : Toutes les actions importantes sont tracées et consultables à tout moment.

      Architecture Technique

      L’architecture de l’interface principale s’appuie sur plusieurs composants principaux :

      • LoadReferences : Gère la détection et la sélection dynamique des versions TIA Portal installées, via la méthode InitializeApp et le scan du répertoire Siemens Automation.

      • UserControl : Chaque module métier (diagnostic, réception de projet, etc.) est encapsulé dans un UserControl, intégré dynamiquement dans l’interface principale.

      • SessionManager : Assure la gestion des sessions, droits utilisateurs, et l’adaptation de l’interface à chaque profil.

      • Logger : Centralise l’historique des actions, accessible depuis l’interface.

      D’autres composants assurent la gestion de la personnalisation (logo, couleurs), la sauvegarde des préférences et l’extension modulaire de l’application.

      Interface Utilisateur

      L’interface utilisateur est organisée en plusieurs zones :

      • Menu principal : Rétractable/étirable, permet la navigation rapide entre les modules et l’accès aux fonctions avancées (admin, terminal, logs).

      • Zone centrale : Affiche le module métier sélectionné (diagnostic, réception de projet, etc.), sous forme de UserControl.

      • Barre supérieure : Informations sur la session, options de connexion/déconnexion, gestion de la fenêtre.

      • Zone de logs : Historique des actions, accessible à tout moment.

      L’ergonomie a été pensée pour maximiser la productivité, avec une gestion intuitive des fenêtres et une adaptation automatique de l’affichage selon le profil utilisateur.

      Prérequis et Compatibilité

      Pour utiliser l’interface principale, il faut :

      • Que l’application AgroFlow soit installée sur le poste de travail

      • Que TIA Portal (V17 minimum) soit installé, avec éventuellement plusieurs versions détectées automatiquement

      • Accès aux références internes Agro Mousquetaires (bibliothèque LTU, standards de projet)

      Guide d'Utilisation

      Étapes principales d’utilisation de l’interface :

      1. Sélection de la version TIA Portal : Au démarrage, choisissez la version à utiliser (le choix est mémorisé).

      2. Connexion utilisateur ou admin : Selon le profil, accédez aux fonctions adaptées.

      3. Navigation entre les modules : Sélectionnez la fonction souhaitée (diagnostic, réception de projet, etc.) via le menu principal.

      4. Utilisation des modules : Chaque module s’ouvre dans la zone centrale, avec gestion des logs et navigation simplifiée.

      5. Consultation des logs : Suivez l’historique des actions et opérations réalisées via l’interface.

      Extension et Personnalisation

      L’interface principale est conçue pour être évolutive et personnalisable :

      • Ajout facile de nouveaux modules (UserControls)

      • Gestion centralisée des droits et profils utilisateurs

      • Personnalisation des couleurs, logos, messages conformément à la charte Agro Mousquetaires
